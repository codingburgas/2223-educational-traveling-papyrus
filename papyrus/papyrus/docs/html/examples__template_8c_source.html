1/*
2 WELCOME raylib EXAMPLES CONTRIBUTOR!
3
4 This is a basic template to anyone ready to contribute with some code example for the library,
5 here there are some guidelines on how to create an example to be included in raylib
6
7 1. File naming: <module>_<description> - Lower case filename, words separated by underscore,
8 no more than 3-4 words in total to describe the example. <module> referes to the primary
9 raylib module the example is more related with (code, shapes, textures, models, shaders, raudio).
10 i.e: core_input_multitouch, shapes_lines_bezier, shaders_palette_switch
11
12 2. Follow below template structure, example info should list the module, the short description
13 and the author of the example, twitter or github info could be also provided for the author.
14 Short description should also be used on the title of the window.
15
16 3. Code should be organized by sections:[Initialization]- [Update] - [Draw] - [De-Initialization]
17 Place your code between the dotted lines for every section, please don't mix update logic with drawing
18 and remember to unload all loaded resources.
19
20 4. Code should follow raylib conventions: https://github.com/raysan5/raylib/wiki/raylib-coding-conventions
21 Try to be very organized, using line-breaks appropiately.
22
23 5. Add comments to the specific parts of code the example is focus on.
24 Don't abuse with comments, try to be clear and impersonal on the comments.
25
26 6. Try to keep the example simple, under 300 code lines if possible. Try to avoid external dependencies.
27 Try to avoid defining functions outside the main(). Example should be as self-contained as possible.
28
29 7. About external resources, they should be placed in a [resources] folder and those resources
30 should be open and free for use and distribution. Avoid propietary content.
31
32 8. Try to keep the example simple but with a creative touch.
33 Simple but beautiful examples are more appealing to users!
34
35 9. In case of additional information is required, just come to raylib Discord channel: example-contributions
36
37 10. Have fun!
38*/
